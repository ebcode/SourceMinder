## Changes Made

### `perl/perl_language.c`

#### `func0op_call_expression` handler (new)
Zero-argument built-ins like `wantarray`, `time`, `localtime` parse as
`func0op_call_expression` in tree-sitter-perl. Added:
- `func0op_call_expression` field to `perl_symbols` struct
- Registration in `init_perl_symbols` (length 23)
- `handle_func0op_call`: extracts the node's own text as the function name,
  emits CALL if it passes the filter
- Dispatch in `visit_node` after `func1op_call_expression`

The initial implementation iterated named children; corrected to extract node text
directly since the keyword token is an anonymous (unnamed) tree-sitter node.

#### `func1op_call_expression` handler (prior session, carried forward)
Single-arg built-ins (`close`, `chomp`, etc.) were already handled. The `close($fh)`
filehandle detection was confirmed working via `file-io.pl` indexing.

### `perl/config/keywords.txt`

- Removed `wantarray` — it is a semantic function (context detection), not syntax.
  Keeping it filtered prevented it from being indexed as CALL.

### `shared/column_schema.def`

Updated `-d` / `--definition` help text to clarify:
- `-d` alone shows the D column without filtering
- `-d 0` / `-d 1` filter to usages or definitions respectively
- Updated example to show all three forms

### `query-index.c`

Added three lines to the **Quick Start** section of `--help`:
```
qi user --def               # Only definitions (where 'user' is declared)
qi user --usage             # Only usages (call sites, references)
qi user -d                  # Show D column (1=def, 0=usage) without filtering
```

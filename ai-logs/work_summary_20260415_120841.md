## Session A: Perl Indexer

### 1. `self` Removed from Keywords

`self` is not a Perl keyword — it's just a convention. Removed from
`perl/config/keywords.txt` so that `$self` is correctly indexed as ARG/VAR throughout OOP
code.

### 2. `phaser_statement` Handler (BEGIN / END / INIT / CHECK / UNITCHECK)

**Problem:** `BEGIN { ... }` and `END { ... }` blocks were not indexed at all, and code
inside them was invisible to qi.

**Investigation:** tree-sitter-perl produces a `phaser_statement` node. Its first child is
the keyword node (BEGIN, END, etc.), and `ts_node_type()` on that child returns the phaser
name directly as a C string.

**Fix:**
- Added `phaser_statement` to the symbols struct and `init_perl_symbols` (length `16` —
  critical: an earlier typo of `17` caused `ts_language_symbol_for_name` to return 0 and
  produced no matches).
- Added `handle_phaser_statement`: reads `ts_node_type(keyword)` for the name,
  `format_source_location` for the location, emits as `CONTEXT_FUNCTION` with `definition="1"`,
  then calls `process_children` to recurse into the block body.
- Dispatched in `visit_node` after `handle_anonymous_sub`.

---

### 3. `container_variable` Gap

**Problem:** Direct hash/array element access like `$ENV{APP_ENV}` uses a `container_variable`
node (not `scalar`). These were silently dropped — neither indexed nor recursed into — so
everything inside BEGIN blocks using `$ENV{...}` was invisible.

**Fix:**
- Added `container_variable` to the symbols struct and init.
- Added `container_variable` to the standalone sigil branch in `visit_node` so it gets
  passed to `index_sigil_node` as `CONTEXT_VARIABLE`.
- Also added `container_variable` to the complex-dereference inner-sigil check in
  `index_sigil_node`.

---

### 4. `hash_element_expression` Handler — Parent Column for PROP

**Problem:** `$self->{name}` was not indexing the key `name` as `CONTEXT_PROPERTY`, and
no `parent` column was populated for property accesses.

**Investigation:** Reviewed `c_language.c` and `php_language.c` to understand the parent
column pattern before implementing.

**Fix:**
- Added `hash_element_expression` to symbols struct and init (length `23`).
- Added `extract_sigil_varname` helper: walks children of a sigil node to find the
  `varname` child, extracts its text, returns `true` if `len > 1`.
- Added `handle_hash_element_expression`: iterates children:
  - `scalar` or `container_variable` → extract container name as `parent_name`, index as
    `CONTEXT_VARIABLE` with `definition="0"`
  - `autoquoted_bareword` → index as `CONTEXT_PROPERTY` with `definition="0"` and
    `.parent = parent_name`
  - anything else → recurse via `visit_node`
- Dispatched in `visit_node` **before** the standalone scalar/container_variable branch
  (order matters — `hash_element_expression` must intercept first).
- Added `handle_autoquoted_bareword` for barewords appearing outside
  `hash_element_expression` (e.g., in `bless { key => val }` or fat-comma lists):
  indexes as `CONTEXT_PROPERTY` with `definition="0"`.

---

### 5. Sigil TYPE Column

**Problem:** The `TYPE` column was empty for all Perl variables — no way to distinguish
`$scalar`, `@array`, and `%hash` in query results.

**Fix:**
- Added three constants to `perl_language.h`:
  ```c
  #define PERL_TYPE_SCALAR  "scalar"
  #define PERL_TYPE_ARRAY   "array"
  #define PERL_TYPE_HASH    "hash"
  ```
- In `index_sigil_node`: after extracting the varname, derive `sigil_type` from
  `ts_node_symbol(node)` — `array` → `PERL_TYPE_ARRAY`, `hash` → `PERL_TYPE_HASH`,
  everything else (scalar, container_variable) → `PERL_TYPE_SCALAR`. Pass as `.type` in
  `ExtColumns`.

---

### 6. Complex Dereference Bug (`@{$ref}`)

**Problem:** `@{$list_ref}` — the `array` node's `varname` child contained a nested
`scalar` node. `safe_extract_node_text` on the varname span returned `$list_ref` with the
sigil included, producing a bogus symbol.

**Fix:** In `index_sigil_node`, before extracting varname text: check
`ts_node_child_count(child) > 0`. If the varname has children, it contains a nested sigil
— recursively call `index_sigil_node` on the inner `scalar`/`array`/`hash`/
`container_variable` child instead of extracting raw text.

---

### 7. Modern Perl Signatures

**Problem:** `sub foo($a, $b = 0, @rest)` — parameters inside `signature` nodes were not
indexed as ARG.

**Investigation:** AST shows three parameter node types: `mandatory_parameter`,
`optional_parameter`, `slurpy_parameter`. Each wraps a sigil child plus optional default
expression.

**Fix:**
- Added all three parameter symbols to struct and init.
- Added `handle_parameter`: iterates children; `scalar`/`array`/`hash` → `index_sigil_node`
  as `CONTEXT_ARGUMENT` with `definition="1"`; anything else → `visit_node` (so default
  values are recursed into).
- Dispatched in `visit_node` before `subroutine_declaration_statement`.

---

### 8. Dynamic Method Call Bug (`$self->$orig()`)

**Problem:** `$self->$orig()` — `handle_method_call` called `safe_extract_node_text`
directly on the `method` node. When the method contained a nested `scalar` child, this
returned `$orig` with the sigil, producing a bogus CALL entry.

**Fix:** In `handle_method_call`, when processing the `method` node: check
`ts_node_child_count`. If a `scalar` child is found, dig into its `varname` grandchild
and extract text from there. Fall through to direct extraction only if no scalar child
found.

## Session B: qi `--raw` Flag

### Problem

qi's normal output wraps source lines with:
- `"Searching for: ..."` header
- Table header row
- Per-file headers
- Line number prefixes (`35:`) on every source line
- `--` separators before/after each match block
- Summary stats footer

To use qi output as an `Edit old_string` argument, all of that had to be stripped manually.

### Design

`--raw` suppresses every non-source byte, leaving only bare file lines:
- No "Searching for:" header block (include/exclude/file/within filter lines also suppressed)
- No table header
- No per-file headers
- No line number prefixes on context or expansion lines
- No `--` separators (opening and closing)
- No ANSI color codes
- No summary stats

Used with `-e` (expand full definition) or `-B`/`-A`/`-C` (context lines).

### Implementation

**`shared/file_utils.h`** — added `int raw` to `print_lines_range` signature.

**`shared/file_utils.c`** — `print_lines_range`: when `raw=1`, skip `printf("--\n")`
opener; print each line as `printf("%s", line)` with no prefix and no column trimming.

**`query-index.c`**:
- `print_context_lines` — added `int raw`; skip `--\n` opener; when raw, skip color
  highlighting path and print `printf("%s", line)` directly.
- `print_expansion_or_context` — added `int raw`; pass through to both callees; skip
  closing `--\n` separators when raw.
- `print_results_by_file` — added `int raw`; skip table header, per-file header, row
  output, and summary stats when raw. (Still tracks `current_file` and resets
  `current_file_count` manually for correct `--limit-per-file` behavior.)
- `main()` — added `int raw_mode = 0`; parse `--raw`; wrapped entire "Searching for:"
  header block (all filter echo lines + X-Macro filter printers + within-filter line)
  in `if (!raw_mode)`; pass `raw_mode` to `print_results_by_file`.

### Documentation

- `query-index.c` help text: added 3-line description after `--debug`.
- `README.md` Quick Reference table: appended `--raw` row.

### Verification


## Issues Encountered

### No `::` structure in Perl AST

- **Symptom:** AST explorer showed `MyApp::Config`, `Scalar::Util`, `SUPER::new` as single leaf nodes — no children for `::` segments
- **Root Cause:** The Perl grammar's `_bareword` regex `((::)|([a-zA-Z_]\w*))+` tokenizes `::` as part of the bareword, unlike PHP's grammar which has explicit `namespace_name → name` children separated by `\`
- **Fix:** Cannot follow PHP's AST-walking pattern for namespace extraction. Instead, string-level splitting with known-package tracking is needed
- **Regression Risk:** LOW — no code was changed, only investigation
- **Follow-Up:** Implement the known-package tracking approach

## Notes / Design Decisions

- **Why not always split on `::`:** `SUPER::new` in `$class->SUPER::new(%args)` uses `SUPER` as a pseudo-package that dispatches to the parent class. There is no `package SUPER;` declaration, so unconditional splitting would produce a bogus namespace. The known-package guard avoids this.
- **PHP vs Perl namespace AST:** PHP's grammar produces structured `namespace_name` nodes with child `name` tokens separated by `\`. Perl's grammar treats `::` as an inline regex alternative within `_bareword`. These are fundamentally different designs — PHP enables AST-walking, Perl requires string splitting.
- **Scope of tracking:** Package tracking needs to be per-file (reset at each `parser_parse_file` call). A simple dynamic array or hash set of known package prefixes would suffice.

## 2. `shift`-Style Parameter Extraction ‚Üí CONTEXT_ARGUMENT

**Problem:** `my $self = shift` was indexed as VAR, not ARG. The existing `@_` detection
only handled the list form `my ($self) = @_`.

**Investigation:** Used `ast-explorer-perl` on a new `tools/sources/perl/shift-unshift.pl`
test file. Found that `shift` produces a `func1op_call_expression` node (one of tree-sitter-perl's
built-in unary operator nodes). Discussed `func0op_call_expression` (zero-arg builtins) and
`func1op_call_expression` (one-arg builtins like shift, pop, chomp, length, etc.) ‚Äî concluded
no general CALL handler is warranted since these are all language keywords.

**Decision:** `shift` and `unshift` stay in `perl/config/keywords.txt`. Removing them is
not necessary for ARG detection; the detection is purely structural.

**Fix:** Added `func1op_call_expression` to the symbols struct and `init_perl_symbols`.
Extended `handle_variable_declaration` to detect a second ARG pattern alongside the existing
`@_` check:
- Case 1 (existing): `my ($self) = @_` ‚Äî RHS is `array` with varname `_`
- Case 2 (new): `my $self = shift` ‚Äî RHS is `func1op_call_expression` where the `function`
  field is "shift" AND there is no explicit non-`@_` array target

`my $item = shift @$queue` (explicit non-`@_` target) correctly remains VAR.

**Verified:** `qi % -f shift-unshift.pl -i arg` returns 12 ARG entries including `class`,
`host`, `port`, `user` from successive shifts. `self` absent ‚Äî expected, it's in
`perl/config/keywords.txt`.

## 3. `use constant` Constants ‚Üí CONTEXT_VARIABLE with modifier "const"

**Problem:** `use constant PI => 3.14` and `use constant { MAX_RETRIES => 3, ... }` did not
index the constant names.

**Investigation:** AST shows two forms:
- Single: `use_statement ‚Üí list_expression ‚Üí autoquoted_bareword "PI"`
- Block: `use_statement ‚Üí anonymous_hash_expression ‚Üí list_expression ‚Üí autoquoted_bareword`
  (nested list_expressions for multi-constant blocks)

Key insight: `=>` (fat comma) auto-quotes the left side, so constant names are always
`autoquoted_bareword` nodes. Values are `number`, `string_literal`, or regular `bareword` ‚Äî
never `autoquoted_bareword` ‚Äî so a recursive search for `autoquoted_bareword` is safe.

**Fix:**
- Added `anonymous_hash_expression` to symbols struct and init
- Added `index_constant_names()` helper ‚Äî recursively descends looking for `autoquoted_bareword`,
  emits each as `CONTEXT_VARIABLE` with `modifier = "const"`, `definition = "1"`
- Extended `handle_use_statement`: after indexing the module name, if `module_name == "constant"`,
  call `index_constant_names` on all subsequent children

**Verified:** `qi PI MAX_RETRIES TIMEOUT BASE_URL -f imports.pl -v` shows all four with
`MOD=const`, `D=1`, `CTX=VAR`.

## 4. Goto and Label Handlers

**New test file:** `tools/sources/perl/goto.pl` ‚Äî labels with `last`/`next`/`redo`,
`goto LABEL`, `goto &func` tail calls, `goto $expr`.

**Investigation:** AST reveals:
- `statement_label` ‚Äî contains `identifier` child (the label name) + the labeled statement
- `goto_expression` ‚Äî contains either `label` (goto LABEL), `function_call_expression`
  (goto &func), or `scalar` (goto $expr ‚Äî skip)
- `loopex_expression` ‚Äî `last`/`next`/`redo`, optionally with `label` child

**Fix:** Added 5 new symbols (`statement_label`, `goto_expression`, `loopex_expression`,
`identifier`, `label`) to struct and init. Added three handlers:
- `handle_statement_label` ‚Äî indexes `identifier` child as `CONTEXT_LABEL` (`D=1`),
  then recurses via `process_children` into the labeled statement
- `handle_goto_expression` ‚Äî `label` child ‚Üí `CONTEXT_GOTO` (`D=0`); `function_call_expression`
  child ‚Üí extracts `varname` from function field ‚Üí `CONTEXT_GOTO` (`D=0`); scalar ‚Üí skip
- `handle_loopex_expression` ‚Äî `label` child ‚Üí `CONTEXT_GOTO` (`D=0`)

All three dispatched in `visit_node` before the TODO comment.

**Verified:** `qi % -f goto.pl -x noise` shows `OUTER`, `INNER`, `RETRY`, `LOOP`, `INVALID`
as LABEL; all `last`/`next`/`redo`/`goto` references as GOTO; `goto &func` targets
(`handle_request`, `handle_event`) as GOTO without `&` prefix.

## 6. Export List Indexing ‚Üí CONTEXT_EXPORT

**Problem:** Names inside `our @EXPORT = qw(trim slugify is_empty)` were not indexed as
exports. They were findable as FUNC (since they're also defined as subs) but `qi trim -i exp`
returned nothing.

**Fix:** Extended `handle_variable_declaration` to detect the export pattern:
- After ARG detection, check: `modifier == "our"` AND array varname is `"EXPORT"` or `"EXPORT_OK"`
- If so, find `quoted_word_list` in parent `assignment_expression`
- Split `string_content` on whitespace, emit each name as `CONTEXT_EXPORT` with `D=1`
- Line numbers tracked per-word by counting newlines within the string_content

`@EXPORT` / `@EXPORT_OK` arrays themselves remain indexed as VAR (correct ‚Äî they are variables).

**Verified:** `qi trim slugify is_empty -f exports.pl -i exp` returns all three at lines
10, 11, 12 exactly.

## Next Steps

- **`special-blocks.pl` AST exploration** ‚Äî `BEGIN`, `END`, `DESTROY`, `AUTOLOAD` not yet
  explored. `BEGIN`/`END` likely produce `phaser_statement` or similar nodes in tree-sitter-perl.
  `AUTOLOAD` is just a sub with a special name; `DESTROY` likewise. `$AUTOLOAD` variable
  may need special handling.
- **`CONTEXT_PROPERTY`** ‚Äî `$self->{name}` hash element access is ubiquitous in Perl OOP.
  The AST shows `hash_element_expression` nodes. Indexing the key as PROP would make
  `qi name -i prop` useful for finding OOP attribute access patterns.
- **`EXPORT_OK` names** ‚Äî currently only the qw() form is handled. Bare list form
  `@EXPORT_OK = ('foo', 'bar')` (array of string literals) is not indexed as EXPORT.
- **`use constant` with bareword values** ‚Äî `use constant FOUR => TWO + TWO` where value
  references another constant; edge case, low priority.
- **Anonymous sub ‚Üí FUNC for LHS variable** ‚Äî `my $greet = sub { ... }` ‚Äî the variable
  `greet` is indexed as VAR; could also emit as FUNC when RHS is `anonymous_subroutine_expression`.
- **`signatures.pl`** ‚Äî modern Perl sub signatures (`sub foo($a, $b)`) not yet explored.
- **`references.pl` remaining** ‚Äî `refgen_expression` (`\&foo`), anon array/hash constructors.

We also added `comment_utils.h`, three new stopword removals, and expanded the `perl_symbols` struct from 17 to 28 fields.

## 1. Comment Indexing (`handle_comment`, `handle_pod`)

### `handle_comment`
- Indexes words from Perl `# ...` comment nodes as `CONTEXT_COMMENT`
- Uses `strip_comment_delimiters()` from `shared/comment_utils.h` (already handles `#`)
- Added shebang guard: `if (comment_text[0] == '#' && comment_text[1] == '!')` ÔÇö skips `#!/usr/bin/perl` entirely rather than indexing `!/usr/bin/perl` as a garbage token

### `handle_pod`
- Indexes words from `=pod ... =cut` documentation blocks as `CONTEXT_COMMENT`
- Directive lines (`=head1`, `=head2`, `=pod`, `=cut`, etc.): skips the directive keyword itself, but indexes any trailing words on the same line (e.g. `=head1 NAME` indexes `NAME`)
- **Line number attribution:** all POD words carry their actual source line number, computed by counting `\n` characters as an offset from the tree-sitter node start point (`int current_line = line; ... current_line++` on each newline advance). This was a two-step fix ÔÇö initially all POD words landed on the `=pod` start line.
- POD verbatim/code blocks (indented lines) are intentionally left indexed ÔÇö they are real references to symbols and `-x noise` filters them when unwanted.

### Added include
- `#include "../shared/comment_utils.h"` added to `perl_language.c`

## 2. Import Indexing (`handle_use_statement`, `handle_require_expression`)

### `handle_use_statement`
- Handles `use Module::Name;` and `use Module qw(name1 name2);`
- Indexes the `package` child (module name) as `CONTEXT_IMPORT`
- Walks `quoted_word_list` Ôćĺ `string_content`, splits on whitespace, indexes each imported name as `CONTEXT_IMPORT`
- `use_version_statement` (e.g. `use 5.010`) produces nothing ÔÇö no handler, children have no useful content

### `handle_require_expression`
- Handles `require Module::Name` and `require "File.pm"`
- Finds `autoquoted_bareword`, `bareword`, or `string_literal` child Ôćĺ indexes as `CONTEXT_IMPORT`
- `eval { require Module }` handled automatically ÔÇö fallthrough `process_children` recurses into the `require_expression` inside the `block`

### New symbols
`package_name`, `require_expression`, `autoquoted_bareword`, `quoted_word_list`

### Key AST insight
In `use_statement`, the keyword `use` is an anonymous node. `ts_language_symbol_for_name(..., true)` (named-only) does not match it. The first named `package` child is the module name ÔÇö no offset counting needed.

## 3. Package Indexing (`handle_package_statement`)

- Indexes `package MyApp::Config;` Ôćĺ `MyApp::Config` as `CONTEXT_NAMESPACE`
- **Bug encountered:** Initial implementation counted to `pkg_count == 2` to skip the keyword, but the `package` keyword is an anonymous node and doesn't match `perl_symbols.package_name` (named=true). The first match is always the module name. Fixed to take the first match directly.

## 4. Subroutine Indexing (`handle_subroutine_declaration`)

- Indexes `sub foo { ... }` Ôćĺ `foo` (the `bareword` child) as `CONTEXT_FUNCTION`
- Calls `process_children` after indexing the name to recurse into the block body (variables, comments, nested calls all indexed)
- `new` was previously filtered as a stopword ÔÇö removed from `keywords.txt` this session

---

## 5. Method Call Indexing (`handle_method_call`)

- Indexes `$obj->method_name()` Ôćĺ `method_name` (the `method` child) as `CONTEXT_CALL`
- Handles `SUPER::new` style calls naturally (tree-sitter gives the full text as one `method` node)
- Calls `process_children` to catch nested method calls in arguments

### New symbols
`method_call_expression`, `method`

---

## 6. Function Call Indexing (`handle_function_call`)

- Handles both `function_call_expression` and `ambiguous_function_call_expression`
- Indexes the `function` child as `CONTEXT_CALL`, then recurses
- Covers Moose DSL (`has`, `extends`, `around`, `requires`), builtins (`sprintf`, `push`, `print`), and user-defined functions called without `->` or `&`

### Design decision: `has` as `CALL` not `PROPERTY`
- `has 'name' => (...)` in Moose declares an attribute. Considered indexing the string argument as `CONTEXT_PROPERTY`, but rejected: a programmer using `has` as a regular function name would see unexpected PROPERTY entries with no explanation. Consistent treatment as CALL is clearer and unsurprising.

### Stopword removals
- `has` removed from `shared/config/stopwords.txt`
- `new` removed from `c/config/keywords.txt` (Perl constructor)

## Next Steps

- **`signatures.pl`:** Modern Perl subroutine signatures (`sub foo($a, $b) { ... }`) ÔÇö likely a different AST node than `variable_declaration`; needs `ast-explorer-perl` investigation
- **`references.pl`:** Code references (`\&foo`, `$coderef->()`), anonymous subs assigned to variables
- **`anonymous.pl`:** Anonymous subs (`sub { ... }`), closures ÔÇö no `bareword` child for the name
- **`regex.pl`:** Regex patterns ÔÇö probably not worth indexing deeply, but regex variable captures (`$1`, `$2`) and named captures (`(?<name>...)`) are interesting
- **`modern-perl.pl`:** `say`, `state`, `given/when`, `feature` pragmas
- **Argument indexing:** Sub parameters (e.g. `my ($self, $name) = @_`) are currently indexed as VAR, but not distinguished as `CONTEXT_ARGUMENT`. C/Python use CONTEXT_ARGUMENT for parameters ÔÇö Perl could too, but requires knowing we're inside a sub's parameter destructuring
- **`use constant` constants:** `use constant PI => 3.14` ÔÇö `PI` is currently not indexed; could be `CONTEXT_VARIABLE` with `const` modifier
- **Anonymous sub indexing:** `my $greet = sub { ... }` ÔÇö the variable is indexed but not as FUNC; consider indexing the LHS variable as FUNC when RHS is an anonymous sub

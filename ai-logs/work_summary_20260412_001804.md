# Work Summary: Perl Indexer — Consistency Pass & Feature Additions
Timestamp: 2026-04-12 00:18:04

## Overview

This session resumed Perl indexer work. We started with a brief review of the prior session's work summary, explored the current state of `references.pl`, then took a step back to audit `perl_language.c` for consistency with the other language indexers (C, PHP, Python). Seven distinct improvements were made.

---

## 1. `coderef_call_expression` Handler

**Problem:** `$greet->("World")` was indexing `greet` as VAR at the call site.

**Fix:** Added `handle_coderef_call` which:
- Finds the leading `scalar` child, extracts its `varname`, indexes as `CONTEXT_CALL`
- Skips recursing into that scalar (prevents double-indexing as VAR)
- Recurses into the argument children normally

**New symbol:** `coderef_call_expression` (length 23 — caught a length-22 bug during debugging)

**Dispatched in** `visit_node` after `function_call_expression`.

**Also required:** `#include <stdbool.h>` added to `perl_language.c`

**Verified:** `greet` → CALL on line 23, `func` → CALL on line 35 of references.pl.

---

## 2. `is_definition` Column — All Handlers

**Problem:** Every `add_entry` call in Perl used `NO_EXTENSIBLE_COLUMNS`, so the `D` (is_definition) column was always NULL. `qi foo --def` and `qi foo --usage` did not work for Perl.

**Fix:** Updated all 7 `add_entry` call sites across 5 handlers:

| Handler | Context | `definition` |
|---|---|---|
| `handle_subroutine_declaration` | FUNC | `"1"` |
| `handle_package_statement` | NAMESPACE | `"1"` |
| `handle_method_call` | CALL | `"0"` |
| `handle_function_call` | CALL | `"0"` |
| `handle_coderef_call` | CALL | `"0"` |
| `handle_use_statement` (×2) | IMP | `"0"` |
| `handle_require_expression` | IMP | `"0"` |

---

## 3. `format_source_location` for Subroutine Definitions

**Problem:** `qi sub_name -e` (expand) did not work for Perl subroutines — the location field was `NULL` so the indexer had no byte range to expand.

**Fix:** In `handle_subroutine_declaration`, added:
```c
char location[128];
format_source_location(node, location, sizeof(location));
```
…and passed `location` (instead of `NULL`) to `add_entry`.

`string_utils.h` was already included. No new dependencies needed.

**Verified:** `qi apply_to_list -f references.pl -e` expands the full sub body.

---

## 4. String Literal Indexing (`handle_string`)

**Problem:** No string words were indexed as `CONTEXT_STRING` for Perl. `qi symbol -i string` returned nothing; `-x noise` had nothing to filter. `string_content` was registered in the symbols struct but never used.

**Fix:**
- Added `interpolated_string_literal` to the symbols struct and `init_perl_symbols`
- Added `handle_string` (matching C/Python pattern): iterates `string_content` children, splits on whitespace, cleans with `filter_clean_string_symbol`, indexes as `CONTEXT_STRING`
- Calls `process_children` at the end so interpolated variable references inside `"..."` strings are also indexed (as VAR/CALL)
- Dispatched both `string_literal` and `interpolated_string_literal` in `visit_node`

**Verified:** `qi % -f references.pl -i string` returns `val`, `localhost`, `Hello`, `World`, `parent`, `child`.

---

## 5. Anonymous Subroutines as `<lambda>` (`handle_anonymous_sub`)

**Problem:** `my $greet = sub { ... }` had no handler — the anonymous sub was invisible to `qi -e` and `--def`. PHP and Python both index anonymous functions as `<lambda>` with `CONTEXT_LAMBDA`.

**Fix:**
- Added `anonymous_subroutine_expression` to symbols struct and `init_perl_symbols` (length 31)
- Added `handle_anonymous_sub`: calls `format_source_location`, emits `<lambda>` as `CONTEXT_LAMBDA` with `definition = "1"`, then recurses into block body
- Dispatched in `visit_node` after `subroutine_declaration_statement`

**Verified:** `qi '<lambda>' -f references.pl -e` shows all 6 anonymous subs with correct line ranges expanded.

---

## 6. Parent Column for Method Calls

**Problem:** `$obj->method()` indexed `method` as CALL with no `.parent`. `qi method -p obj` did not work for Perl. PHP and C both populate the parent column for member/method access.

**Fix:** Rewrote `handle_method_call` to gather both invocant and method name before calling `add_entry`:
- `scalar` invocant (`$obj->method()`): extracts `varname` from the scalar child
- `bareword` invocant (`Dog->new()`): extracts bareword text directly
- Sets `.parent = invocant` on the CALL entry

**Verified:** `qi % -f oop-classic.pl -i call -v` shows PAR = `self`, `class`, `Dog`, `dog` correctly.

---

## 7. Parameters as `CONTEXT_ARGUMENT` (not VAR)

**Problem:** `my ($self, $name) = @_` was indexed as variable declarations (VAR), not as function parameters (ARG). C and Python both emit `CONTEXT_ARGUMENT` for function parameters.

**Fix:**
- Added `ContextType ctx` parameter to `index_sigil_node` (was hardcoded to `CONTEXT_VARIABLE`)
- Updated both call sites to pass the context
- In `handle_variable_declaration`, added `@_` detection: calls `ts_node_parent()` on the `variable_declaration` node, checks if the parent `assignment_expression` has an `array` sibling with `varname = "_"`, and if so sets `ctx = CONTEXT_ARGUMENT`

**Note:** `shift`-style parameter extraction (`my $self = shift`) is not yet handled — no test files used this pattern.

**Verified:** `qi % -f oop-classic.pl -i arg` returns `class`, `args`, `trick`. Broader test shows 34 ARG entries across all Perl test files, with VAR count reduced correspondingly.

## qi Tool Evaluation

This session dogfooded `qi` in place of Read/Grep/Search tools. Key findings:

**Strengths:** `--toc` for file orientation, `-e` expand for reading implementations, multi-symbol search, context type filtering (`-i func`, `-x noise`), `D` and PAR columns for verification.

**Weaknesses:** Struct fields not indexed (had to fall back to Read for `perl_symbols` struct body); no way to extract raw source text for edit anchors (qi locates, Read must supply the `old_string`); large `-v` output is token-expensive in context-limited sessions.

**Missing feature:** A `--raw` or `--copy` mode that outputs exact source text suitable for use as an edit anchor would close the loop between "find it" and "edit it."

**Verdict:** Recommended for semantic navigation and verification. Best used alongside Read for editing workflows. The `--def` / `--usage` / `--within` / `-p` features would be particularly valuable when navigating an unfamiliar codebase.

## Next Steps

- **`shift`-style parameter extraction:** `my $self = shift` inside a sub body should also produce ARG entries. Requires detecting `function_call_expression` with function = `shift` on the RHS of an assignment.
- **`use constant` constants:** `use constant PI => 3.14` — `PI` is not currently indexed. Could be `CONTEXT_VARIABLE` with `modifier = "const"`.
- **Anonymous sub → FUNC:** `my $greet = sub { ... }` — the variable `greet` is currently indexed as VAR. Consider also indexing it as FUNC when the RHS is an `anonymous_subroutine_expression`.
- **`signatures.pl`:** Modern Perl subroutine signatures (`sub foo($a, $b) { ... }`) — likely a different AST node; needs `ast-explorer-perl` investigation.
- **`references.pl`:** Remaining items — `refgen_expression` (`\&foo`), anonymous array/hash constructors.
- **Chained method calls:** `__PACKAGE__->meta->make_immutable` — the intermediate invocant is the result of a prior call, not a named variable. Currently `make_immutable` gets no parent. Hard problem, low priority.
- **Debug logging:** Add `g_debug` instrumentation to each Perl handler (currently only in `visit_node`). Makes `--debug` mode useful for tracing Perl parse issues.
- **Struct field indexing in qi:** The C indexer doesn't appear to pick up `TSSymbol`-typed struct fields as PROP. Worth investigating whether this is a type annotation issue or a broader gap.

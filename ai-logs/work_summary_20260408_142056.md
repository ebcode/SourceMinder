## Bug Fix: `$data` in Return Statement Not Indexed

**Problem:** `return $data;` on line 34 of `variables.pl` was not being indexed. The `visit_node()` dispatcher only handled `variable_declaration` nodes â€” standalone `scalar`/`array`/`hash` nodes (like `$data` inside a `return_expression`) were reached by recursion but never indexed.

**Root Cause:** The AST for `return $data;` is:
```
return_expression
  return
  scalar
    $
    varname "data"
```
The `scalar` node here is not inside a `variable_declaration`, so it was silently skipped.

**Fix:** 
- Added a `definition` parameter to `index_sigil_node()` to distinguish declarations (`"1"`) from references (`"0"`)
- Added a dispatch case in `visit_node()` for standalone `scalar`/`array`/`hash` nodes, indexing them as variable references
